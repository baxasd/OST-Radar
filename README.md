# OST Radar Pipeline

Radar Extension of my project OST

Developed with Python ZeroMQ for high performance file writing and analysis

## Architecture

The system is split into three distinct tools:

1. **`stream.py` (The Source)**
   A headless Python process that writes RDHM to disk and streams data over tcp port with the help of ZeroMQ

2. **`monitor.py` (The Viewer)**
   The visual interface. Connects to streamer and visualizes doppler heatmap

3. **`analyze.py` (The Cruncher)**
   The post-run processing script. It loads the massive `.parquet` files generated by the stream, runs the signal processing to extract cadence metrics, and exports the final summary and timeseries CSVs.

## Hardware Requirements
* Texas Instruments mmWave Radar (Configured for 60GHz FMCW)
* Two USB connections:
  * **CLI Port** (115200 baud) for sending configuration commands
  * **DATA Port** (921600 baud) for the high-speed binary stream

## Usage

**1. Start the Hardware Stream**
Plug in the radar and run the streamer. It will auto-detect the USB ports.
```bash
python stream.py

```

**2. Open the Live Monitor**
Open a new terminal (or go to another computer on the same Wi-Fi network) and launch the monitor:

## Data Format

Radar frames are chunked and saved using PyArrow/Parquet for memory efficiency. Each row contains:

* `timestamp`: Standard Unix epoch time.
* `rdhm_bytes`: The raw, flat `uint16` bytes of the 2D Range-Doppler Heat Map matrix. Radar configuration metadata is embedded directly into the Parquet file's schema metadata.
